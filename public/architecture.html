JWT Auth Flow
1
Client POSTs credentials to /api/auth/login
2
Server verifies bcrypt hash, issues HS256 JWT (24h TTL) with userId, orgId, orgRole, plan
3
Token stored in Zustand ai-governance-auth → persisted to localStorage
4
useApi() hook reads live Zustand token, attaches Authorization: Bearer <token> to every request
5
withAuth() wrapper on every API route verifies token, checks role hierarchy, injects user + organizationId
6
Returns 401 (no/bad token), 403 (insufficient role), 402 (plan limit exceeded)
7
Logout clears Zustand store + localStorage + auth_token cookie

Infrastructure Architecture
Vercel (hosting + CDN) · Neon Serverless PostgreSQL · Redis · Vercel Blob (files) · Resend (email) · GitHub CI/CD
Infrastructure Components
▲ Vercel — Hosting & CDN
Runtime: Node.js (serverless functions)
Build: prisma generate → prisma db push → next build
Max Duration: 60s (AI routes), 10s (standard)
Deploy trigger: Git push to main → auto-deploy
CDN: Global edge network
Domain: aigovernancetower.com
Public static: /public/ (demo.html, architecture.html)
🐘 Neon — Serverless PostgreSQL
Provider: Neon (serverless Postgres)
Connection: DATABASE_URL
ORM: Prisma v5.14 (type-safe, compiled)
Binary targets: native + rhel-openssl-1.0.x + rhel-openssl-3.0.x
Scale: Serverless, auto-scale to zero
Tables: 37 models, multi-tenant (orgId row-level)
Migrations: prisma db push (schema-first)
⚡ Redis — Cache Layer
Library: ioredis v5.4.1
Connection: REDIS_URL
Config: maxRetries=3, lazyConnect, no readyCheck
TTL default: 300 seconds (5 minutes)
Key pattern: org:{orgId}:{key}
Degradation: Non-fatal; cache miss → DB fallback
Rate limiting: RATE_LIMIT_MAX / RATE_LIMIT_WINDOW_MS
📦 Vercel Blob — File Storage
Package: @vercel/blob v2.3.3
Usage: Evidence file uploads (ISO 42001 controls, risk assessments, data assets)
Model: EvidenceFile.url → Blob URL
API route: POST /api/upload (multipart form)
Polymorphic: controlId | riskId | modelId | dataAssetId
Access: Public CDN URLs, no auth required on file
📧 Resend — Email Service
Package: resend v6.12.0
API Key: RESEND_API_KEY
Base URL: NEXT_PUBLIC_APP_URL
Use cases: Email verification, org invitations, scheduled report delivery
From: enquiry@aigovernancetower.com
🔒 Security Layer
Auth: JWT HS256 (jose), 24h TTL
Passwords: bcryptjs hash
CORS: Next.js default (same-origin)
Headers: CSP, HSTS, X-Frame-Options: DENY, X-XSS-Protection, X-Content-Type-Options
Rate limit: Middleware (Redis-backed)
Multi-tenant isolation: orgId on every DB query
📡 GitHub → CI/CD Pipeline
Repo: github.com/forprashantthakur/ai-governance-tower
Branch: main → auto-deploy to Vercel
Build cmd: prisma generate && prisma db push --accept-data-loss && next build
Type check: tsc --noEmit (pre-commit)
Deploy time: ~60–90 seconds
📊 Observability (Optional)
LLM tracing: Langfuse (cloud.langfuse.com)
Keys: LANGFUSE_SECRET_KEY, LANGFUSE_PUBLIC_KEY, LANGFUSE_HOST
Tracks: AI prompt logs, token usage, latency, toxicity scores
Health: GET /api/health endpoint

Integration Architecture
Anthropic Claude API · n8n Webhooks · Resend Email · Vercel Blob · Langfuse (optional) · JWT Auth
🤖
Anthropic Claude API
Model: claude-haiku-4-5-20251001 · SDK: @anthropic-ai/sdk ^0.90.0
Integration 1 — AI Use Case Finder POST /api/maturity-assessment
Max tokens: 8,000 · Role: "Principal AI Strategy Consultant"
Input: 30+ discovery fields (org profile, industry, KPI baseline/target, pain points, data readiness, revenue levers, existing systems, regulatory context, business goals)
Output: JSON — use_case_name, priority_score (0–100), priority_tier, score_breakdown (7 dimensions), business_problem, ai_solution, expected_business_impact, agentic_ai_design (agents + decision_flow), n8n_workflow (nodes, connections, integrations), implementation_plan (phases + timeline), integration_architecture (systems, data_flow, API requirements)
Stored in: MaturityAssessment.useCases (JSON column)
Audit action: AI_MATURITY_ASSESSMENT_COMPLETED
Client 5-step form → withAuth + Zod validate → buildPrompt() → Claude API (60s max) → Parse JSON → Save + Audit → 201 Response
Integration 2 — n8n Workflow Builder POST /api/n8n-workflow
Max tokens: 6,000 · Role: "Expert n8n workflow engineer"
Input: description, triggerType (webhook|schedule|manual), integrations[], complexity (simple|medium|complex)
Output: Full n8n-compatible export JSON — nodes (with typeVersions for n8n 1.x), connections map, credentials_required, setup_steps, test_instructions
Node types supported: 40+ — manualTrigger, webhook, scheduleTrigger, httpRequest, if, switch, set, code, emailSend, slack, postgres, mongo, redis, salesforce, hubspot, @n8n/langchain.agent, lmChatAnthropic, lmChatOpenAi, awsS3, googleDrive…
Import: n8n_export JSON is directly importable into n8n instance
Builder form → buildPrompt() → Claude API (60s) → Strip markdown fences → Build n8nExport → 200 + Download
🔄
n8n Outbound Webhooks
Project event triggers → n8n workflows · Async, non-blocking · Per-project configuration
Trigger Events (N8nTriggerEvent enum):
PHASE_COMPLETE  MILESTONE_REACHED  TASK_DONE  RISK_THRESHOLD  EXPERIMENT_LOGGED
Payload: event, projectId, projectName, timestamp, context object, optional custom template (N8nWebhook.payloadTemplate)
Timeout: 10 seconds per webhook
Result capture: lastTriggeredAt, lastStatus ("success"|"error"), lastResponse (JSON) stored in N8nWebhook model
Webhook lifecycle:
Project event fires→fireWebhooks(projectId, event, ctx)
Fetch active webhooks for project→Build payload
POST to webhookUrl→Promise.allSettled (non-blocking)
Capture status + response → DB→Manual trigger: POST /api/projects/[id]/webhooks/[whid]/trigger
